# 👁️ Panopticon

Panopticon（パノプティコン）は、M5StickS3と M5Stack赤外線送受信ユニット（U002）を用いて、あらゆる赤外線信号を傍受・記録し、またHome Assistantと連携して自在に信号を発射するための「究極のIR（赤外線）マルチツール・ファームウェア」です。

Home Assistant（HA）と双方向に連携し、スマートホームの赤外線ゲートウェイとして機能するだけでなく、単独で持ち運んで未知の赤外線信号（家電量販店のテレビやホテルのエアコンなど）を収集し、後で一括ダウンロードする「シギント（Signals Intelligence）」機能も備えています。

---

## 🛠️ ハードウェア要件

- **M5StickS3**
- **M5Stack U002 IR Unit** (赤外線送受信ユニット)
  - 接続ポート: Groveポート
  - 使用ピン: `G9` (TX/送信), `G10` (RX/受信)

## ✨ 主な機能とアプリケーション

起動すると、本体の液晶ディスプレイに「PANOPTICON OS」のメインメニューが表示されます。側面ボタン（BtnB）でスクロールし、正面ボタン（BtnA）で以下の3つの専用アプリを起動します。

### 1. 📡 Dumb Pipe
Home Assistant との間で赤外線信号を土管（Dumb Pipe）のように中継する基本モードです。
- **TX（送信）:** HAから `/tx` エンドポイントに送られてきたJSON配列（生のパルス長）を受信し、即座に赤外線を空間へ放射します。
- **RX（受信）:** 空間の赤外線信号を傍受し、それをデコードして美しいHEX文字列（例: `AEHA 0x...`）として画面に表示。同時に、生パルス配列をJSON化してHAのWebhookへPOSTします。
- **ステルスモード:** 稼働中に正面ボタン（BtnA）を押すと、画面のバックライトが完全に消灯し、暗闇に溶け込みながら省電力で中継を続けます。

### 2. 🎯 Sniper
狙ったタイミングで確実に赤外線を発射するための「現地点検・テスト用」モードです。
- HAから `/tx` で信号が送られてくると、即座に発射するのではなく、一旦内部の「薬室（Chamber）」に装填（ロード）され、画面が `[ LOADED ]` と赤く表示されます。
- 同時に、HAに対して `Target_Locked` イベントをWebhookで通知します。
- 現地にいるユーザーが、機器に向けて正面ボタン（BtnA）を押した瞬間に、装填された信号が発射され、画面がフラッシュして空になります。

### 3. 🕵️ Sigint Log (Signals Intelligence)
Wi-Fiの届かない場所（外出先や量販店など）で、未知の赤外線リモコンの信号を傍受し、本体のフラッシュメモリに密かに記録し続けるロガーモードです。
- **ローカル保存:** 受信した信号のプロトコル、HEX文字列、生パルス配列、タイムスタンプをJSON Lines (`.txt`) 形式で `LittleFS` に追記保存します。
- **安全な検証サイクル:** 信号を受信した後、正面ボタン（BtnA）を短押しするとその場で同じ信号を発射してテストできます。もし「これは違う」と思えば、正面ボタンを長押し（DEL）することで、メモリ上のバッファから即座に破棄され、ファイルには保存されません。
- **一括回収と消去:** 自宅のWi-Fi環境に戻った後、PCやスマホのブラウザから `http://[PanopticonのIP]:8080/logs` にアクセスすると、収集したログファイルの一覧が表示されます。ワンクリックで生データをダウンロードでき、赤いボタンを押せば本体のメモリを完全に消去（Clear All）できます。

## 🔧 その他の強力な機能

### 🎵 Find My Panopticon
「Panopticonを部屋のどこに置いたか忘れた！」という時のための機能です。
ブラウザから `http://[PanopticonのIP]:8080/beep` にアクセスすると、本体のスピーカーから「ピピッ... ピピッ...」と高いビープ音が非同期で鳴り続け、居場所を主張します。本体のいずれかのボタンを押すか、`/found` にアクセスすると鳴り止みます。

### 🧠 Smart Custom Decoders & 物理ノイズフィルター
標準の `IRremoteESP8266` ライブラリでは `UNKNOWN`（未知）として破棄されてしまう、日本の複雑なエアコンの長大信号（AEHA）や、特殊なタイミングを持つシーリングライト（SwitchBot / NEC-L）の信号を、独自アルゴリズムで完璧にデコードします。
最大256ビット以上の超長大コードでも上位ビットが欠落することなく、`AEHA 0x1234567890ABCDEF... (88bit)` のような美しいHEX文字列に復元します。
また、本体を物理的に叩いた時のチャタリングノイズをソフトウェアレベルで100%弾く、強力なスケルチ（ノイズフィルター）を備えています。

## 🚀 セットアップ

1. リポジトリの `secrets.ini.example` を `secrets.ini` にリネームします。
2. `secrets.ini` 内に、ご自身のWi-Fiの SSID と パスワードを記述します。
   ```ini
   [secrets]
   wifi_ssid = "YOUR_WIFI_SSID"
   wifi_password = "YOUR_WIFI_PASSWORD"
   ```
3. `src/Config.h` を開き、必要に応じて `WEBHOOK_URL` をご自身のHome Assistantの環境に合わせて変更します。
4. PlatformIO (VSCode) を使用して、M5StickS3 に書き込みます。

### 🔌 API エンドポイント一覧
- `POST /tx` : HAからJSON形式（`{"raw": [9000, 4500, ...]}`）でパルス配列を受け取り、Dumb Pipe または Sniper へ送ります。
- `GET /logs` : 保存されたSigintログファイルの一覧をHTMLで表示します。
- `GET /download?file=/sigint_xxx.txt` : 指定したログファイルをブラウザに強制ダウンロードさせます。
- `POST /logs/clear` : 本体に保存されたすべてのログファイルを消去します。
- `GET /beep` : 本体からビープ音を鳴らし続けます。
- `GET /found` : ビープ音を停止します。

## 💡 アーキテクチャの美学
このファームウェアは、極限までシンプルかつ堅牢な「イベント・プッシュ型」かつ「プラグイン・アーキテクチャ」を採用しています。
C++のグローバルインスタンスの初期化順序（SIOF）による割り込み暴走（Interrupt Storm）を完全に防ぐため、すべてのハードウェアタイマーやオブジェクトはOSと電位が安定した後に遅延初期化（Lazy Initialization）されます。
新しいアプリを追加したい場合は、`AppInterface` を継承したクラスを作成し、`main.cpp` の配列に `push_back` するだけで自動的にメニューに組み込まれ、安全にルーティングされます。
