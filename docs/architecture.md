# アーキテクチャ解説 (Architecture)

IR-Trigger の設計思想と構造について解説します。

---

## 1. ハブ＆スポーク構造

Home Assistant は物理的な送信機（USBドングルやIRハブ）と、実際に操作する家電（テレビ、エアコン）を明確に分けて管理します。

- **Transmitter (Hub)**: `local_usb` や `esphome` として定義される「物理的な送受信デバイス」。
- **Device (Spoke)**: `transmitter` プロパティによって特定のハブに紐付けられた仮想的な家電。
- **Entity**: 家電ごとにボタンエンティティが生成されます。

この構造により、HAのデバイスレジストリ上で「このテレビはこのドングルから操作される」という論理的な接続が可視化されます。

---

## 2. 動的ルーティングエンジン

IR信号を受信した際のフローは以下の通りです：

1. **信号検知**: 受信機がコード（例: `NEC_1234`）を検知。
2. **正規化**: `IR-Trigger.yaml` の辞書から「どのデバイスの、どのボタンか」を特定。
3. **モード判定**:
   - `mode_entity` で指定された状態（例: "Movie", "Game"）と、常に有効な "always" モードを確認。
4. **アクション実行**:
   - **Repeat**: 対象デバイスが `repeat` リストにあれば、送信機から即座に再送。
   - **Bind**: 入力ソースとターゲットがバインドされていれば、ターゲットの対応するコードを送信。

---

## 3. 無限ループ防止 (Loop Prevention)

赤外線リピーターとして動作させる際、自ら送信した赤外線を自らの受信機で検知し、無限ループに陥る可能性があります。
`LocalUSBTransmitter` に `local_receivers` を指定することで、特定の受信機からの入力を一定時間（または特定条件で）無視し、ループを遮断します。
