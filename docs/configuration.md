# 詳細設定ガイド (Configuration Guide)

`IR-Trigger.yaml` を使用したマルチ状態管理および高度な設定方法について解説します。

---

## 1. 全体構造

```yaml
transmitters: ...   # 送信機の定義
devices: ...        # 家電デバイスの定義
global: ...         # グローバル設定（常に有効）
state_machines: ... # ステートマシン（モード別の動的ルーティング）
```

---

## 2. グローバル設定 (`global`)

常に有効な動作を定義します。

| キー | 型 | 内容 |
| :--- | :--- | :--- |
| `repeat` | list | 受信時に自動で再送（リピート）するデバイス ID のリスト |
| `remap` | dict | 特定のコードに対するグローバルなアクション（最優先・排他的） |

---

## 3. ステートマシン (`state_machines`)

複数の独立した状態管理マシンを定義できます。

| キー | 型 | 内容 |
| :--- | :--- | :--- |
| `name` | string | (任意) ステートマシンの識別名 |
| `mode_entity` | string | 状態判定に使用する HA エンティティ ID |
| `modes` | dict | モード名とルーティング定義のマップ |

### モード内設定 (`modes`)
各モード内で `remap` と `bind` を定義できます。

*   **`remap`**: 特定のコードに対するアクション定義。
*   **`bind`**: ソース（リモコン）とターゲット（家電）のボタン一括紐付け。

**評価順序:** `remap` -> `bind` の順で検索され、最初にマッチしたものが実行されます。その時点でそのステートマシンの評価は終了しますが、他のステートマシンの評価は継続されます。

---

## 4. 評価フローと排他制御

信号受信時のルーティングエンジンは以下の順序で動作します。

1.  **Repeat**: `global.repeat` をチェックし、該当すればリピート送信（独立して動作）。
2.  **Global Remap**: `global.remap` をチェック。マッチすれば実行し、**以降の全評価を中断（排他）**。
3.  **State Machines**: 定義された全ステートマシンを順番に評価。マッチした時点でそのマシンの評価を終了し、次のマシンへ。

---

## 5. 設定の反映
`IR-Trigger.yaml` や辞書ファイルを編集した後は、Home Assistant のサービス `ir_trigger.reload` を実行することで即座に反映できます。
